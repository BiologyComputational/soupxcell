🏫 The Setup

You have 3,639 students (cells) from 4 different schools (donors). Each student takes a test with 80,712 questions (loci). But instead of choosing A or B once, each question is answered by counting reads — like recording every time a student whispers "A" or "B" while thinking.

⚠️ Why so sparse?

Each student only answered about 50 of 80,712 questions — because each cell only produces reads at a tiny fraction of all SNP positions. Most cells at most loci: zero reads = NaN (unanswered). This sparsity matters for every metric later.

🔬 The key biological fact

Students do NOT choose their answer freely. Each student already has a pre-filled answer sheet inside them based on their DNA. We are just reading it — imperfectly — by sampling reads. The reads are noisy observations, not choices.

The Data — Two Sparse Matrices

ref.mtx and alt.mtx — The Raw Ingredients

vartrix produces two matrices by scanning each cell's reads at each SNP position. These are the only inputs to the correction formula.

ref.mtx — "A" vote counts → R[l,n]

Rows = loci, Columns = cells. Each entry = reference allele read count. Hover cells to see counts.

Blue = has reads. Grey = zero (no coverage at this position for this cell).

alt.mtx — "B" vote counts → A[l,n]

Identical dimensions to ref.mtx. Every row aligns exactly to the same locus. Must be used together.

Green = has reads. Sparse just like ref.mtx — most cells have zero alt reads at most positions.

// Dimensions of both matrices: R[l,n] → 80,712 loci × 3,639 cells ← 293,710,368 entries total A[l,n] → 80,712 loci × 3,639 cells ← same structure // Non-zero entries (covered) = 1,444,860 = 0.49% of all entries // Zero/NaN entries = 292,265,508 = 99.51% sparse
💡 Why we store two matrices instead of one

We need both the "A" count and the "B" count separately so we can compute the ratio of B to total, not just "which allele." Later we will need both numerator and denominator to compute the allele fraction.

Step 1 — Allele Fraction X[l,n]

Converting Raw Votes Into a 0–1 Signal

The first operation: for every covered (non-zero) cell-locus pair, compute what fraction of reads supported the alternative allele.

X[l,n] = A[l,n] / ( R[l,n] + A[l,n] ) // A[l,n] = "B" votes · R[l,n] = "A" votes · denominator = total votes // X = NaN when R[l,n] + A[l,n] = 0 ← student didn't answer this question

X = 0.0

Only "A" reads. Student is reference-homozygous (AA) at this position. No alternative allele.

R=4, A=0
X = 0/(4+0) = 0.000

X ≈ 0.5

Roughly equal "A" and "B" reads. Student is heterozygous (AB). Reads split roughly 50/50.

R=3, A=3
X = 3/(3+3) = 0.500

X = 1.0

Only "B" reads. Student is alternative-homozygous (BB) at this position.

R=0, A=5
X = 5/(0+5) = 1.000

🎮 Interactive: See contamination in action

This cell is AA (reference-homozygous). Its true X should be 0.000. But ambient RNA adds fake "B" reads. Drag to see the effect:

Real example from this run

// Cell: AAACATACCACTCC-1 // Locus: 5847 R[5847, cell] = 3 ← 3 "A" reads (real) A[5847, cell] = 1 ← 1 "B" read ← CONTAMINATION X[5847, cell] = 1 / (3 + 1) = 0.2500 // This cell is AA. True value = 0.000. // The 0.25 is ENTIRELY from ambient RNA. // One rogue read from another donor's fragment.
🔑 Critical insight

Cells do not choose their allele — they contain it. A "B" read appearing in an AA cell means that read came from someone else's floating RNA in the solution, not from the cell itself. That is ambient contamination.

The Problem — Ambient RNA Contamination

The Room Is Always Whispering "B"

When cells are processed in a liquid, some cells lyse (break open) and release their RNA into the surrounding solution. Every droplet then absorbs a small amount of this background "soup". In a 4-donor pool, this soup carries the allele fractions of all four donors mixed.

🧫
Cell (droplet)
// The contamination mixture model: X[l,n] = (1 − α) × X_true[l,n] + α × μ[l] // Where: α = ambient fraction = 0.01262 ← 1.262% of reads are ambient μ[l] = population mean allele fraction at locus l X_true = what the cell actually contains X = what we observe (contaminated)
🎙️ Story: The Room's Voice

The room has a background chatter: at each question l, the room says "B" with probability μ[l]. A fraction α = 1.262% of everything we hear is the room, not the student. So even a pure "A" student picks up some "B" chatter at questions where the room is loud.

What contamination does to donors

The 4 donors cluster by their allele profiles. Ambient RNA pulls every cell toward the population average μ[l], blurring the differences between donors.

Each cluster represents one donor. With contamination, cluster boundaries blur (shown schematically).

Step 2 — The Soup Vector μ[l]

What Does the Room Sound Like at Each Question?

Before correcting, we need to estimate what the background "soup" looks like at every locus. This is the soup vector μ[l].

μ[l] = (1 / |S|) × Σ X[l,n] for all n ∈ S // S = set of singlet cells only (3,523 cells — doublets excluded) // μ[l] = mean allele fraction at locus l across all clean singlet cells // = what the "population" looks like at this question // Then the soup contribution at locus l: soup[l] = α × μ[l] = 0.01262 × μ[l]

Three example loci

// Locus l₁ — "B" rare in pool μ[l₁] = 0.412 → soup = 0.01262×0.412 = 0.00520 // Locus l₂ — "B" very common in pool μ[l₂] = 0.800 → soup = 0.01262×0.800 = 0.01010 // Locus l₃ — "B" rare μ[l₃] = 0.100 → soup = 0.01262×0.100 = 0.00126 // Global mean across all 64,496 loci: mean(soup) = α × mean(μ) = 0.005201 max(soup) = α × max(μ) = 0.006089
🎧 Why exclude doublets from μ?

Doublets already contain a mix of two donors. Including them would make the "room sound" appear noisier than it actually is — like recording background noise while two people are having a conversation in the room. So only singlets (clean cells) contribute to μ[l].

Soup vector μ[l] — distribution across 64,496 loci

Most loci have μ near 0 (reference-dominant pool). A few have high μ (alternate common). This shape determines the OLS slope = 0.3875 later.

💡 μ[l] tells you two things

Direction: where contamination pulls each entry (toward μ[l]).
Magnitude: soup[l] = α × μ[l] is exactly how much B signal gets added to every cell at locus l.

Step 3 — The Correction Formula (Core Equation)

Rearranging the Mixture Model to Recover the Truth

Given the contamination model, we rearrange algebraically to isolate the true signal:

// The mixture model (what we observe): X[l,n] = (1 − α) × X_true[l,n] + α × μ[l] // Rearrange to isolate X_true: X_true = ( X[l,n] − α × μ[l] ) / (1 − α) // Since α = 1.262% ≪ 1, denominator (1−α) = 0.9874 ≈ 1 → drop it: X_corr[l,n] = max( 0, X[l,n] − α × μ[l] ) // max(0,·) = floor: prevents negative allele fractions (physically impossible) // Error from dropping denominator = 1.262% — negligible at this α level
"Take what we observed, subtract the room's contribution, clip at zero."

Worked example — step by step

// Cell: AAACATACCACTCC-1 Locus: 5847 // True genotype = AA → X_true should = 0.000 X_obs = 0.2500 ← contaminated value μ[l] = 0.9800 ← most donors show B here soup = 0.01262 × 0.980 = 0.01237 X_corr = max(0, 0.2500 − 0.01237) = max(0, 0.2376) = 0.2376 // Still not 0.000 — because 0.25 has more B signal // than just the soup. But 0.01237 of fake B is removed.

Step 4 — The Floor: Why 44.32% Entries Hit Zero

When the Entire Observed Signal Is Explained by Noise

An entry is floored to zero when the observed value is already below the expected soup contribution at that locus. This means the cell has no real signal there — what we saw was entirely from ambient RNA.

// Flooring condition: X[l,n] < α × μ[l] → X_corr[l,n] = 0 // In plain terms: "What we heard is weaker than what the room alone would produce." // So the student has NO real B signal at this question. // Numbers: Total covered entries: 1,444,860 Floored to zero: 640,425 = 44.32% Surviving (real signal): 804,435 = 55.68%
"In 44% of cases, every B vote we observed was entirely explained by background noise — the student contributed nothing."

Per-donor floor % (nearly identical → proves global model)

⚖️ Uniformity is the proof

If the ambient model were donor-specific or biased, different donors would have very different floor percentages. The < 0.2 pp variation (44.19%–44.38%) proves the ambient RNA truly comes from a global pool affecting all donors equally.

❗ Flooring ≠ data loss

Setting 44% to zero is NOT deleting data. It is correctly identifying entries where the cell has no detectable alternative allele. These were entirely ambient noise and should be zero. This is the correct biological inference.

📊 The surviving 56%

These entries have X[l,n] stronger than the room noise. They carry real biological signal: cells that are heterozygous (X ≈ 0.5) or alternative-homozygous (X ≈ 1.0) at that locus. These are the entries that drive donor clustering.

Step 5 — Silhouette Coefficient

Does Each Student Sound More Like Their Own Club?

The silhouette coefficient measures cluster quality: is each cell closer to its own donor group than to others?

// For each cell i: s(i) = ( b(i) − a(i) ) / max( a(i), b(i) ) // a(i) = mean Euclidean distance to other cells in SAME donor cluster // b(i) = mean Euclidean distance to cells in NEAREST OTHER cluster // Range [-1, +1] · >0 = closer to own club · <0 = closer to another club // Our results: Global silhouette BEFORE: −0.002560 AFTER: −0.002563 Δ = −0.000003

Per-donor silhouette (hover for formula)

❓ Why is silhouette NEGATIVE?

With 99.51% sparsity, two cells from the same donor typically have reads at completely different loci. Their Euclidean distance is large because most positions are incomparable. Two cells from different donors have the same problem. Distances become essentially random — no systematic separation in Euclidean space.

Silhouette failing here is NOT a bug. It means Euclidean distance is the wrong metric for sparse allele-fraction data.

Silhouette intuition — 4 clubs in feature space
🔢 Worked numerical example

Cell from Donor 0:
a(i) ≈ 0.412 (mean distance to other Donor 0 cells)
b(i) ≈ 0.409 (mean distance to nearest other donor)
s(i) = (0.409 − 0.412) / 0.412 = −0.0073

The distances are nearly equal because sparsity makes all pairs look similarly distant. Global mean −0.002560 ≈ this value.

🎓 Q from conversation: "Are we trying to CORRECT silhouette? What does Sil before/after mean?"

No — soupXcell is not trying to improve the silhouette score. The "Sil before" and "Sil after" values are a purely a diagnostic check, not a correction target.

We are correcting the allele fraction data. After correcting the data, we re-run silhouette on the same cluster labels to ask: "Did cleaning the data accidentally break or destabilise the clusters?" The answer here is no — Δ = −0.000003, effectively zero.

Think of it like: you clean a messy spreadsheet (data correction), then re-run your analysis (silhouette) to confirm the cleaning did not corrupt the results. Silhouette Δ ≈ 0 confirms it did not — the cluster structure is intact. The real improvement signal is in within-cluster standard deviation (Δ = −0.000887).

Step 6 — Davies-Bouldin Index

A Better Lens: Club Tidiness vs Club Separation

Because silhouette fails under sparsity, we also use Davies-Bouldin — which compares cluster summaries instead of every pairwise distance.

// DB = (1/K) × Σₖ max_{j≠k} (sₖ + sⱼ) / d(cₖ, cⱼ) // sₖ = scatter of cluster k = mean distance of cells to cluster centroid // d(cₖ,cⱼ) = Euclidean distance between centroids of clusters k and j // Lower DB = tighter, more separated clusters = better DB_before: 1.9618 DB_after: 1.9588 Δ = −0.0030 (improvement)

Silhouette vs Davies-Bouldin

MetricWhat it asksVulnerable toOur result

🎭 Story: why DB survives sparsity

Silhouette asks: "Does student S1 sound more like their own club than any other specific student from Club 2?" — impossible when S1 and their clubmates answer different questions.

DB asks: "Is the average voice of Club A different from Club B?" — much easier because averaging across many students smooths the gaps.

Why DB improves when silhouette does not

After correction, each cell's allele fractions have a small noise component removed (0.005201 units). This makes cells within the same cluster slightly more consistent:

// Scatter sₖ depends on within-cluster variance // Correction removes α×μ[l] noise → sₖ decreases slightly Within-cluster std: 0.204753 → 0.203866 Δ = −0.000887 (−0.43%) // This is the TRUE signal of correction working // (silhouette cannot detect this but DB can)
💡 Key takeaway for the professor

The DB improvement (+0.15%) and within-cluster std decrease (−0.43%) are the correct metrics for evaluating ambient RNA correction. Silhouette Δ ≈ 0 is not a failure — it tells us something even more interesting, which we explain in Step 12.

🎓 Q from conversation: "Is soupXcell re-clustering cells after correction?"

No — soupXcell never re-clusters. The 4 donor assignments (Cluster 0–3) from souporcell are fixed and never changed. soupXcell only changes the allele fraction values in the matrix.

What happens:

① Clusters are assigned by souporcell → fixed.
② soupXcell corrects X[l,n] → values get smaller / some go to 0.
③ Silhouette and DB are re-computed using the same cluster labels on the new values.
④ We compare before vs after to see if the existing clusters became more coherent.

If you wanted to re-cluster, you would have to feed X_corrected.mtx back into souporcell — that is a separate downstream experiment, not part of soupXcell's pipeline.

Module 3 — Dimensionality Reduction: PCA → t-SNE / UMAP

Seeing the 4 Donors in 2D — Before and After Correction

After correcting the allele fractions we want to see whether the four donor clusters are separable. But the data is 64,496-dimensional — one dimension per locus. No human can see 64,496 dimensions. Dimensionality reduction solves this: compress those 64,496 numbers per cell into just 2 numbers you can plot.

🏫 Story: why we need this

Each student (cell) has a 64,496-length answer sheet. Imagine trying to figure out which school each student came from just by comparing answer sheets directly — you'd have to look at 64,496 questions per comparison. PCA + t-SNE compress each student into just 2 coordinates (x, y) that capture the most important patterns, so you can plot all 3,522 students on one chart and see the 4 clubs visually.

Step-by-step: why each stage is necessary

① Why PCA first?

64,496 dimensions is too large for t-SNE or UMAP to handle directly — they would be extremely slow and numerically unstable. PCA reduces to 50 "principal components" — the 50 directions of maximum variance in the data — keeping 95%+ of the signal but compressing computation by 1,300×.

// Randomised SVD decomposition: X_qc [3,522 × 64,496] → compute top 50 eigenvectors of covariance matrix → project cells onto those 50 directions → output: [3,522 × 50] (~20s, ~130× compression) // Why 50 components? Captures ~95% of variance. // Trade-off: more components = more signal but slower t-SNE.

② Why t-SNE after PCA?

t-SNE (t-distributed Stochastic Neighbour Embedding) finds a 2D layout that preserves the neighbourhood structure of the 50D PCA space. Cells that are close in 50D PCA space (= genetically similar) are placed close together in the 2D plot. Unlike PCA alone, t-SNE can reveal non-linear cluster structure.

// Barnes-Hut t-SNE (approximate, O(N log N)): Input: [3,522 × 50] (PCA components) Output: [3,522 × 2] (2D layout) Iters: 1,000 Perplexity: 30 Time: ~47s // Perplexity ≈ "effective number of neighbours" // Range 5–50; 30 is standard for scRNA-seq // t-SNE preserves local structure — clusters visible

③ Why also UMAP?

UMAP (Uniform Manifold Approximation and Projection) is much faster than t-SNE (~1s vs ~47s) and better preserves both local and global structure. It has become the standard in modern single-cell analysis. soupXcell computes both so you can compare.

// UMAP (k-NN graph + force-directed layout): Input: [3,522 × 50] (same PCA as t-SNE) Output: [3,522 × 2] (2D layout) Time: ~1 second // UMAP vs t-SNE: UMAP preserves global distances better. // t-SNE: tighter clusters, better local separation. // Both: run on X_raw AND X_corr → 4 total scatter plots

Step 7 — The Simulation Benchmark

Proving the Correction Works — Without Needing Biological Ground Truth

In real data, we can never know X_true exactly. So we run a controlled experiment: inject a known amount of contamination, apply the correction, and measure how much we recovered. Like a noise-cancelling headphone test.

// THREE STEPS — repeated for 6 levels × 5 trials: // Step 1: Inject known contamination at level α_sim X_inj[l,n] = X_raw[l,n] + α_sim × μ[l] ← we know exactly what we added // Step 2: Apply correction using true α = 0.01262 (not α_sim) X_rec[l,n] = max(0, X_inj[l,n] − α × μ[l]) ← same formula as real correction // Step 3: Measure recovery error (RMSE) RMSE = √(mean( (X_inj − α_sim×μ[l] − X_rec)² )) over all covered entries
❓ "But if we add α_sim and remove α, aren't we just undoing ourselves?"

No — this is the key insight. We inject α_sim but subtract α. These are different values. The test checks: "If the true contamination is α_sim but our method assumes α, how large is the residual error?" This tests robustness to mis-specified α — exactly what happens in practice where α is estimated, not known.

🎓 Q from conversation: "What does OLS actually mean here?"

OLS = Ordinary Least Squares. It is the standard method for fitting the best straight line through a set of data points by minimising the sum of squared vertical gaps between each point and the line.

We have 6 data points: x = α_sim (contamination injected), y = RMSE (error measured). OLS finds the slope b such that y = b·x fits best — meaning the total squared error between predicted and actual RMSE values is as small as possible.

Why through the origin (no intercept)? When α_sim = 0, RMSE must be exactly 0 — no contamination means no error. So the line is forced through (0, 0). This simplifies the formula to:
slope = Σ(α_sim × RMSE) / Σ(α_sim²) = 0.15213 / 0.3926 = 0.3875

What the slope 0.3875 tells you: for every 1 extra percentage point of contamination beyond α, the residual error grows by 0.3875 percentage points. The slope equals √(mean(μ[l]²)) — the root-mean-square of the soup vector, a fixed property of this dataset's genetic diversity. A dataset with a stronger alternative allele signal would have a larger slope.

Step 10 — Why Silhouette Δ ≈ 0 Even at 50% Contamination

The Most Important Conceptual Result

The simulation shows silhouette barely changes even at 50% contamination. This is not a failure of correction — it reveals a fundamental mathematical property of uniform shifts.

// After injection + correction, each entry becomes: X_rec[l,n] = X[l,n] + (α_sim − α) × μ[l] // ↑ call this δ_l — same for EVERY cell at locus l // Distance between cell i and cell j at locus l: d_before = |X[l,i] − X[l,j]| d_after = |X[l,i] + δ_l − X[l,j] − δ_l| = |X[l,i] − X[l,j]| ← δ_l CANCELS EXACTLY ✓ // Therefore: a(i) unchanged · b(i) unchanged · s(i) unchanged · ΔSil = 0
"Ambient RNA moves everyone together — so relative distances stay the same, and silhouette sees nothing."
Silhouette Δ across all 6 simulation levels

All values within 10⁻⁴ of zero — the tiny observed Δ is sampling noise from the ≤150 cells/cluster subsample.

🎭 Story: The Volume Knob

Ambient RNA is like turning the volume knob up on every student equally at each question. If Student A was at volume 0.2 and Student B at volume 0.5, and we turn both up by δ = 0.1:

A → 0.3, B → 0.6. Difference is still 0.3. The silhouette only measures differences — the uniform shift is invisible to it.

The Key Insight for the Professor

Donor identity is encoded in which loci show the alternative allele — the binary 0/1 pattern across 64,496 positions. Ambient RNA shifts allele-fraction magnitudes uniformly but never changes which loci are informative for which donor.
