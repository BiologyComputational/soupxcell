Why Dimensionality Reduction? — The core problem
1
64,496 dimensions. Each cell is a point in 64,496-dimensional space — one dimension per QC locus. No human can visualise 64,496 dimensions. We need to compress to 2D to see donor clusters.
2
t-SNE/UMAP can't handle 64K features directly. They are O(N log N) in feature count — running on 64,496 features takes hours. PCA first compresses to 50 components (~20s), then t-SNE/UMAP work on 50 dimensions (~47s).
3
Before vs After correction. Running the same embedding on X_raw and X_corr produces side-by-side plots. The correction should make donor clusters slightly tighter (within-cluster std −0.43%). At α=1.26% the visual change is subtle — the simulation R²=1.000 is the real proof.
