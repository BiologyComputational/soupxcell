Opening · ~3 min· ~3 min in
Write the problem on the board first
3 min

Before any derivation, write this one equation on the top of the board. Tell the professor: "everything I am about to derive comes from fixing this."

✏️
Opening equation:
/* The Problem */
X_observed ≠ X_true

/* The Fix */
X_corr[l,n] = max( 0, X_obs[l,n] − α × μ[l] )

/* Where */
α = ambient RNA fraction = 0.01262
μ[l] = population mean allele fraction at locus l
X_obs = contaminated measurement
X_corr = corrected estimate of X_true
🎯 The core problem this derivation solves
Each cell was measured at thousands of genomic positions. Every measurement is contaminated by ambient RNA floating in the sample liquid. The equation above removes that contamination. Every symbol is derived from first principles and every number verified against real run data.

Foundation · ~4 min· ~7 min in
The Two Input Matrices
4 min

Everything begins with two sparse matrices produced by vartrix, which counts how many reads support each allele at each genomic position for each cell.

✏️
Two input matrices:
Matrix 1 R[l, n]
— reference allele reads
— "how many times cell n
showed the A allele at locus l"

Dimensions:
80,712 × 3,639
Matrix 2 A[l, n]
— alternative allele reads
— "how many times cell n
showed the B allele at locus l"

Dimensions:
80,712 × 3,639
✏️
Dataset dimensions:
L = 80,712 ← total genomic loci (SNP positions)
L_qc = 64,496 ← QC-filtered loci used for metrics
N = 3,639 ← total cells (droplets)
N_s = 3,523 ← singlet cells (one donor)
N_d = 117 ← doublet cells (two donors mixed)
K = 4 ← donor clusters

Total entries: 80,712 × 3,639 = 293,710,368
Non-zero: 1,444,860 (0.49%)
Sparsity: 99.51% ← most entries are NaN
📦 Inputs to soupXcell
R[l,n] and A[l,n] are the only two inputs to the correction algorithm. Everything that follows — the soup vector μ[l], the correction formula, the floor, the simulation — is derived from these two matrices and the single scalar α estimated by souporcell in Stage 8.

Definition · ~4 min· ~11 min in
Define X[l,n] — the Allele Fraction
4 min

Combine the two matrices into a single allele fraction matrix — the fraction of reads supporting the alternative allele.

✏️
Allele fraction definition:
X[l,n] = A[l,n] / ( R[l,n] + A[l,n] )
✏️
Interpretation:
X = 0.0 → only reference reads → genotype AA
X = 0.5 → half reference, half alt → genotype AB
X = 1.0 → only alternative reads → genotype BB
X = NaN → no reads at this locus for this cell
✎ Worked example (real data — write on board)
Cell: AAACATACCACTCC-1 Locus: 5847
R[5847, cell] = 3 A[5847, cell] = 1

X[5847, cell] = 1 / ( 3 + 1 ) = 0.2500

⚠ This cell is AA (X_true = 0.0)
The 0.25 is contamination — 1 of 4 reads
came from ambient RNA of another donor

The Problem · ~5 min· ~16 min in
The Contamination Mixture Model
5 min

When cells are processed in liquid, some cells lyse (break open) and release RNA into the solution. Every droplet captures a small fraction of this background "soup". The observed X is a mixture.

✏️
The contamination mixture model — the core equation:
X_obs[l,n] = (1 − α) · X_true[l,n] + α · μ[l]
✏️
Term definitions:
↑ true signal diluted ↑ ambient added

α = fraction of reads that are ambient = 0.01262
μ[l] = mean allele fraction of the ambient pool at locus l
X_true = the cell's actual genotype signal
X_obs = what we measure (contaminated)
Fig 3 — Contamination mixture: X_obs = (1−α)·X_true + α·μ[l]
Each bar shows observed X decomposed into true signal (green) and soup (amber). At α=1.26%, soup contribution is small but systematic across all 1.44M entries.
🔬 Interpretation of the mixture model
98.74% of every allele fraction measurement is the cell's true genotype signal; 1.26% is ambient RNA contributing its own allele pattern. The ambient pool has a characteristic profile μ[l] — different at every locus. Before the correction can be applied, μ[l] must be estimated from the data — that is the purpose of Step 4.
✏️
Verification with real data:
Cell is AA: X_true = 0.0
μ[5847] = 0.98 (most donors are BB at this locus)

Predicted X_obs = (1 − 0.01262) × 0.0 + 0.01262 × 0.98
= 0 + 0.01237
= 0.01237

But we observed 0.2500 ← stochastic noise from low coverage (only 4 reads)
With more reads, X_obs would approach 0.01237

Estimation · ~5 min· ~21 min in
Deriving μ[l] — The Soup Vector
5 min

We need to estimate μ[l] before we can use the mixture model. We use the mean allele fraction across all singlet cells — but not doublets.

✏️
Soup vector definition:
μ[l] = (1 / |S|) × Σ X[l,n] for all n ∈ S
where S = set of singlet cells only = 3,523 cells

Why exclude doublets?
Doublets = two cells in one droplet = mixed genotype from 2 donors
Including them would artificially inflate the "population average"
→ always use singlets only for μ[l]
✏️
Soup contribution per locus:
soup[l] = α × μ[l]

Three examples (write as a column):
μ[l] = 0.412 → soup = 0.01262 × 0.412 = 0.005199
μ[l] = 0.800 → soup = 0.01262 × 0.800 = 0.010096
μ[l] = 0.100 → soup = 0.01262 × 0.100 = 0.001262

Mean soup across all loci:
mean(soup) = α × mean(μ) = 0.01262 × 0.4122 = 0.005201

Core Derivation · ~6 min· ~27 min in
Deriving the Correction Formula from the Mixture Model
6 min

Now we have the mixture model (Step 3) and μ[l] (Step 4). Rearrange to isolate X_true.

✏️
Correction formula derivation — step by step:
Start:
X_obs = (1 − α) · X_true + α · μ[l]
mixture model from Step 3
→
X_obs − α · μ[l] = (1 − α) · X_true
subtract α·μ[l] from both sides
→
X_true = ( X_obs − α · μ[l] ) / (1 − α)
divide both sides by (1−α)
→
since α = 0.0126 ≪ 1: (1 − α) ≈ 0.9874 ≈ 1
approximation — error < 1.3%
→
X_corr ≈ X_obs − α · μ[l]
drop denominator (≈1)
→
X_corr = max(0, X_obs − α · μ[l] )
floor at 0 (cannot be negative)
0 / 6
╔══ FINAL CORRECTION FORMULA ══╗
X_corr[l,n] = max( 0, X[l,n] − α · μ[l] )
✎ Apply to worked example
X_obs = 0.2500
μ[5847] = 0.9800
soup = 0.01262 × 0.980 = 0.01237

X_corr = max(0, 0.2500 − 0.01237)
= max(0, 0.23763)
= 0.2376

real run
Histogram of |X_raw − X_corr| corrections. Mean = 0.005201 = α×mean(μ). Distribution follows the shape of the soup vector.
Fig 5b — Soup vector μ[l]: distribution across 80,712 loci

Mean correction — the key check
mean|X_obs − X_corr| = 0.005201
α × mean(μ) = 0.01262 × 0.4122 = 0.005201

✓ They match exactly.
The average correction = the average soup contribution.
This confirms the correction is perfectly calibrated.

Consequence · ~4 min· ~31 min in
The Floor — Why 44.32% of Entries Hit Zero
4 min

An entry is floored when its observed value is already below the expected soup contribution. The entire signal is ambient noise.

✏️
Flooring condition:
Entry is floored when: X[l,n] < α × μ[l]

Then: X_corr[l,n] = 0

Physical meaning:
"The observed signal is weaker than what ambient noise alone would produce."
"Therefore: the cell has NO real alternative allele signal here."
✏️
Numbers from this run:
Total covered entries: 1,444,860
Floored to zero: 640,425
Floor percentage: 640,425 / 1,444,860 = 44.32%

Per-donor (nearly identical → proves global model):
Donor 0: 44.19% Donor 1: 44.38%
Donor 2: 44.34% Donor 3: 44.37%

Variation < 0.2 pp ← confirms the ambient model is global, not donor-specific
Fig 6 — Floor: 44.32% of entries collapse to zero per donor
All four donors show ~44.3% floor rate. Near-identical bars prove the ambient model is global — not donor-specific.
Fig 6b — Surviving signal (55.68%) vs floored (44.32%)
Blue = entries with real signal above noise. Red = floored to zero. Flooring is truth recovery, not data loss.
💬 Question from discussion
"44% sounds like we're deleting half the data. Isn't that a problem?"
Flooring to zero is truth recovery. For those 640,425 entries, X_obs < soup[l] — the observed signal is smaller than what ambient noise alone would produce. There is no real alt-allele signal to preserve. Setting X_corr = 0 is the correct biological answer.
✅ What 44.32% floored actually means
Flooring to zero is truth recovery, not data deletion. For these 640,425 entries, the observed allele fraction is smaller than the expected ambient contribution (X_obs < α·μ[l]). The entire observed signal is ambient noise — the cell genuinely has no alternative allele signal at that position. The 55.68% of entries that survive carry real genotype signal above the ambient threshold.

Evaluation · ~5 min· ~36 min in
The Silhouette Coefficient
5 min

Now we need to evaluate whether correction improved the clustering of 4 donor groups. The silhouette measures whether each cell is closer to its own donor cluster than to others.

✏️
Silhouette formula:
s(i) = ( b(i) − a(i) ) / max( a(i), b(i) )
a(i) = mean Euclidean distance from cell i to cells in its OWN cluster
b(i) = mean Euclidean distance from cell i to cells in NEAREST other cluster

Range: −1 ≤ s(i) ≤ +1

s ≈ +1 → well inside own cluster ← ideal
s ≈ 0 → on the boundary
s ≈ −1 → closer to another cluster
✎ Worked example — Donor 0 cell
Data lives in 64,496-dimensional space
Sparsity: 99.51% ← most entries are zero

a(i) ≈ 0.412 (intra-cluster distance, dominated by missing overlap)
b(i) ≈ 0.409 (inter-cluster distance, same problem)

s(i) = (0.409 − 0.412) / max(0.412, 0.409)
= −0.003 / 0.412
= −0.0073

Global mean: −0.002560 (before correction)
−0.002563 (after correction)
Δ = −0.000003 ← effectively zero
Fig 7 — Silhouette per donor: before vs after correction
Donor 3 is the only cluster with positive silhouette (+0.001330) — its genotype is most distinct. All Δ values are within 10⁻⁵ — silhouette cannot detect the uniform correction.
💡 Why negative silhouette is expected here
At 99.51% sparsity, two cells from the same donor typically have reads at completely different loci. Their Euclidean distance is dominated by non-overlapping positions, not by genetic similarity — identical to the situation for two cells from different donors. The result: a(i) ≈ b(i) ≈ 0.41, giving s(i) ≈ −0.003. This is a sparsity effect, not clustering failure. Step 14 gives a formal proof of why silhouette Δ ≈ 0 is mathematically expected after correction.

Evaluation · ~4 min· ~40 min in
Davies-Bouldin Index — A Better Lens
4 min

Because silhouette fails under sparsity, we also compute the Davies-Bouldin index, which compares cluster-level summaries rather than every pairwise cell distance.

✏️
Davies-Bouldin formula:
DB = (1/K) × Σₖ max_{j≠k} ( sₖ + sⱼ ) / d(cₖ, cⱼ)
K = 4 (donor clusters)
sₖ = scatter of cluster k = mean distance of cells to centroid
d(cₖ, cⱼ) = Euclidean distance between centroids k and j

Lower DB = better (tighter clusters, farther apart)

DB_before = 1.9618
DB_after = 1.9588
Δ = −0.0030 (0.15% improvement)

DB improves because correction reduces within-cluster variance (sₖ↓)
Silhouette stays flat because pairwise distances are unaffected
→ DB is the correct metric here; silhouette is not
Fig — Davies-Bouldin: scatter sₖ vs centroid distance d(cₖ,cⱼ)

Validation · ~5 min· ~53 min in
The Simulation Benchmark — Proving Correction Works
5 min

In real data we cannot verify correction because we don't know X_true (inferred from clustering — Step 7). The simulation creates a controlled experiment: inject known contamination, then recover it, then measure the residual.

✏️
The three simulation steps:
// Step 1: Inject known contamination at level α_sim
X_inj[l,n] = X_raw[l,n] + α_sim × μ[l]

// Step 2: Apply correction (using true α, not α_sim)
X_rec[l,n] = max(0, X_inj[l,n] − α × μ[l])

// Step 3: Measure recovery error
RMSE = √( mean( (X_inj − α_sim×μ[l] − X_rec)² ) )
✏️

Key Derivation · ~6 min· ~59 min in
Proving RMSE is Linear in α_sim
6 min

Now we derive algebraically why RMSE must scale linearly with α_sim. This is the most important derivation in the whole model.

✏️
Per-entry error derivation:
error =
X_inj − α_sim×μ[l] − X_rec
definition of error
=
(X_raw + α_sim·μ) − α_sim·μ − max(0, X_raw + α_sim·μ − α·μ)
substitute X_inj and X_rec
=
X_raw − (X_raw + (α_sim−α)·μ)
for non-floored entries, max(0,·)=·
=
(α_sim − α) × μ[l]
X_raw cancels — error depends only on mismatch × μ
0 / 4
✏️
Substituting into RMSE:
RMSE =
√( mean( [ (α_sim−α) × μ[l] ]² ) )
substitute error formula
=
|α_sim−α| × √( mean( μ[l]² ) )
factor out the constant (α_sim−α)
=
|α_sim−α| × √mean(μ²)
√mean(μ²) is a CONSTANT of the dataset
≈
0.3875 × α_sim
since α ≈ 0, treat (α_sim−α) ≈ α_sim
RMSE = √mean(μ[l]²) × (α_sim − α)
slope = 0.3875 = RMS of soup vector · constant for this dataset

The Proof · ~5 min· ~72 min in
R² = 1.000 — Mathematical Exactness
5 min

R² = 1.000 is not "very good fit" — it is algebraically exact. Show the full numerical calculation.

✏️
R² formula and calculation:
R² = 1 − SS_res / SS_tot
mean y =
(0.003873 + 0.019366 + 0.038733 + 0.077471 + 0.116218 + 0.193779) / 6 = 0.074907
mean RMSE across 6 levels
SS_tot =
Σ(RMSE_i − 0.074907)² = 0.025304
total variance in RMSE
SS_res =
Σ(RMSE_i − 0.3875×α_i)² ≈ 3.07 × 10⁻⁹
sum of squared residuals — tiny!
R² =
1 − (3.07×10⁻⁹ / 0.025304) = 1 − 0.000000121 = 0.999999879
≈ 1.000000
R² = 1.000000
The correction formula is the mathematically exact inverse of the contamination model
✅ What R² = 1.000 proves
Every data point lies exactly on the predicted line y = 0.3875·x. Residuals are at 10⁻⁵ scale — the magnitude of floating-point rounding, not model error. R² = 1.000 is an algebraic consequence: for non-floored entries, error = (α_sim − α)·μ[l] exactly, and RMSE = √mean(μ²) × (α_sim − α) exactly. The correction formula is the mathematically exact inverse of the contamination model.

The Key Insight · ~5 min· ~77 min in
Why Silhouette Δ ≈ 0 Even at 50% Contamination
5 min

This is the conceptually richest result. Silhouette does not change because ambient contamination is a uniform translation that cancels in every pairwise distance.

✏️
What the correction does to each matrix entry:
After injection and correction, each entry becomes:
X_rec[l,n] = X[l,n] + ( α_sim − α ) × μ[l]

Let δ_l = (α_sim−α) × μ[l]

Key property: δ_l is the SAME for every cell at locus l
It does not depend on which donor the cell belongs to
💬 Question from discussion
"Why doesn't silhouette change even at 50% contamination? That seems like it should affect everything."
Ambient RNA shifts all cells by δ_l at each locus. In pairwise distances, δ_l appears in both cells and cancels: |X_i + δ_l − X_j − δ_l| = |X_i − X_j|. Silhouette only measures differences between cells — uniform shifts are invisible to it by definition.
✏️
The δ_l cancellation — core proof:
Before:
d(i,j) at locus l = |X[l,i] − X[l,j]|
Euclidean distance
After:
d(i,j) at locus l = |X[l,i] + δ_l − X[l,j] − δ_l|
δ_l added to both cells
=
| X[l,i] − X[l,j] |
δ_l CANCELS EXACTLY
0 / 3
Since every distance is unchanged:
a(i) unchanged → b(i) unchanged → s(i) unchanged

ΔSil = 0 (theoretically exact)

Observed Δ = −0.000003 (real run) is sampling noise only
✏️
The correct sensitivity measure:
Correct sensitivity measure: within-cluster standard deviation

σ_within BEFORE = 0.204753
σ_within AFTER = 0.203866
Δ = −0.000887 (−0.43%)

Correction removes the α×μ[l] variance component
→ This is the true signal that correction is working
🔑 Central insight
"Ambient RNA moves everyone together in the same direction at each locus. When you measure the distance between two people who both moved by the same amount, the distance hasn't changed. Silhouette is a distance metric — it is blind to uniform translations. The within-cluster standard deviation is not. That is why Δ≈0 is a proof, not a failure."

— DATA —
▶ #1 — What exactly is a 'locus'? How is L = 80,712 different from L_qc = 64,496?

A locus (plural: loci) is a specific position in the genome — a single-nucleotide polymorphism (SNP) where different people carry different DNA letters (A vs G, C vs T, etc.). Think of it as one exam question at a fixed position on the chromosome.

L = 80,712 is every SNP position detected by freebayes in the BAM file — all positions where any cell showed any variation.

L_qc = 64,496 is the subset of those positions that souporcell actually used for clustering (Mode B: mapped from cluster_genotypes.vcf). These are higher-confidence SNPs with sufficient coverage and informative variation across donors.

L = 80,712 ← all SNPs detected in BAM L_qc = 64,496 ← high-confidence SNPs used for metrics Difference = 16,216 positions filtered out by QC

The correction (X_corr) is applied to all L = 80,712 loci. The cluster quality metrics (silhouette, DB) are computed only on the L_qc = 64,496 QC loci.

▶ #2 — Why are 99.51% of entries zero/NaN? Is that a problem?

No — it is completely expected in single-cell RNA sequencing. Each cell only produces reads at a small fraction of all SNP positions because:

1. The cell only expresses a subset of genes (most of the genome is not transcribed in any given cell at any given moment).
2. Even at expressed positions, sequencing depth is low — you might get 5–20 reads per cell per expressed region.

So out of 80,712 × 3,639 = 293 million possible (locus, cell) pairs, only 1,444,860 (0.49%) have any reads at all. The rest are genuinely not observed — not missing by error, but simply not measured.

Total entries: 293,710,368 Non-zero: 1,444,860 (0.49%) NaN: 292,265,508 (99.51%)

Why does this cause problems? Euclidean distances between cells become unreliable — two cells from the same donor rarely share covered positions, so their distance looks just as large as between donors. This is why the silhouette is negative.

▶ #3 — What is a doublet, and why does it matter so much?

A doublet is a droplet that accidentally captured two cells instead of one — typically from two different donors. In this run: 117 of 3,639 cells (3.2%) are doublets.

Doublets matter in two ways:

For soup estimation: A doublet's allele fractions look like a 50/50 mix of two donors. Including doublets when computing μ[l] would inflate the "population average" — making the correction subtract too much from everyone. That's why only the 3,523 singlets are used to estimate μ[l].

For clustering: Doublets are excluded from embeddings and metric computations. Donor 0 has 115 doublets (12.5% of its cells) — anomalously high — worth investigating.

Total cells: 3,639 → Singlets: 3,523 + Doublets: 117 Doublets used for μ[l]: NO — excluded from soup estimation Doublets used for embeddings: NO — singlets only (3,522)
— ALLELE FRACTION —
▶ #4 — Why does a cell answer the same question multiple times? I thought it was one answer per question.

This is the most important conceptual shift. A cell does not choose an answer — it contains a pre-determined genetic sequence.

Each read is an independent sampling of that sequence. If a cell has genotype AA at locus l, every read at that position should say "A". If you get 10 reads, you see A ten times: R[l,n] = 10, A[l,n] = 0, X = 0.

Why multiple reads? More reads = more confidence. With 1 read, even noise could look significant. With 100 reads from an AA cell, you'd expect X ≈ 0.00 + tiny noise.

AA cell, 10 reads: R=10, A=0 → X = 0.0 (confident) AA cell, 1 read: R=1, A=0 → X = 0.0 (uncertain) AA cell, 4 reads with contamination: R=3, A=1 → X = 0.25 (noise)

The key insight: contamination hurts most at low coverage. With 4 reads, 1 ambient read = 25% contamination. With 400 reads, 1 ambient read = 0.25% contamination. The correction removes the systematic bias but not individual stochastic noise.

▶ #5 — What does X = 0.5 really mean? Can't a cell be more than 50% heterozygous?

X[l,n] = 0.5 means: at this specific genomic position, the cell carries one copy of the reference allele and one copy of the alternative allele (heterozygous, genotype AB). Humans are diploid — they have exactly two copies of each chromosome.

So at any given SNP position, a cell can only be:

AA: X_true = 0.0 (both copies = reference) AB: X_true = 0.5 (one of each) BB: X_true = 1.0 (both copies = alternative)

There is no "75% heterozygous" at a single SNP. However, across many loci, a donor's overall allele frequency profile creates a complex fingerprint. souporcell uses this fingerprint to separate donors even without knowing their genotypes in advance.

— MIXTURE MODEL —
▶ #6 — What physically is 'ambient RNA' / the 'soup'?

When cells are processed in a microfluidic device (like 10x Genomics Chromium), some cells lyse (burst) before entering a droplet. These broken cells release their RNA into the surrounding liquid — the ambient fluid. Every droplet then captures a small fraction of this free-floating RNA.

The ambient RNA pool is a mixture of RNA from all donors in the sample, roughly in proportion to their cell counts and transcriptional activity. At any locus l, the ambient pool has a characteristic allele frequency μ[l] — the average fraction of alternative allele reads in that pool.

α = 0.01262 means: 1.262% of the RNA molecules captured in a typical droplet came from this ambient pool rather than the captured cell.

α = 0.01262 → 1 in ~79 reads is ambient soup[l] = α × μ[l] → how much ambient signal at locus l This is systematic: every cell in the sample is affected equally
▶ #7 — X_true is never measured. So how do we know the mixture model is correct?

We validate it through the simulation (Step 9). Since we cannot directly measure X_true, we use indirect evidence:

1. Mean correction = mean soup. If the model is correct, the average amount subtracted should equal α × mean(μ). It does: 0.005201 = 0.01262 × 0.4122. ✓

2. Uniform floor% across donors. If the model were wrong and biased toward certain donors, their floor percentages would differ. All four donors: 44.19–44.38%. Variation < 0.2 pp. ✓

3. R² = 1.000 from simulation. We inject known contamination and recover it exactly. An incorrect model would produce non-linearity or drift. The recovery is algebraically exact. ✓

X_true is inferred, not measured. For a Donor 0 cell at a locus where Donor 0 is reference-homozygous, souporcell's clustering tells us X_true = 0. We use that inference to validate the correction — not the other way around.

▶ #8 — Why is α the same for every cell? Surely some cells get more contamination than others?

The model assumes α is a global constant — the same fraction for every droplet. This is a simplification. In reality, different droplets might capture slightly different amounts of ambient RNA depending on their position in the microfluidic device, timing, and other factors.

However, the simplification works because:

1. The variation in α across droplets is small compared to α itself (empirically validated in souporcell's design).
2. Using a single global α makes the correction tractable — estimating per-cell α would require much more data than typically available at low coverage.

The uniformity of floor% across donors (all ≈44.3%) and the R²=1.000 from simulation both confirm that the global α assumption is valid for this dataset.

— CORRECTION FORMULA —
▶ #9 — Why do we drop the (1−α) denominator in the correction? Isn't that wrong?

The exact formula is:

X_true = (X_obs − α·μ[l]) / (1 − α)

soupXcell drops the denominator and uses:

X_corr ≈ X_obs − α·μ[l]

The error introduced is: multiply by (1/(1−α) − 1) = α/(1−α) ≈ α for small α.

At α = 0.01262: error ≈ 1.26% of the corrected value. For a corrected value of 0.50, this is 0.0063 — far smaller than the typical noise from read sampling. The approximation saves computation and avoids division by a number that could be estimated imprecisely.

Exact: X_true = (0.25 − 0.01237) / (1 − 0.01262) = 0.2376 / 0.9874 = 0.2407 Approx: X_corr = 0.25 − 0.01237 = 0.2376 Error: 0.2407 − 0.2376 = 0.0031 (1.3% of corrected value)
▶ #10 — Why is max(0, ·) needed? Can the correction ever go negative?

Yes — it can go negative, and that is physically impossible. An allele fraction must be between 0 and 1. Here is when it happens:

X_obs = 0.005 (small positive observation) α·μ[l] = 0.012 (soup contribution at this locus) X_obs − α·μ[l] = 0.005 − 0.012 = −0.007 ← NEGATIVE → floor to 0

This happens when the observed value is below the expected soup threshold. Physically, it means the entire observed signal — and more — is explained by ambient noise. There is genuinely no real signal here. Setting it to zero is the correct inference: "this cell has no alternative allele at this position."

The max(0,·) is not a hack — it is encoding a biological constraint (allele fractions cannot be negative) and performing implicit hypothesis testing (if X_obs < soup, the signal is entirely noise).

— FLOOR —
▶ #11 — 44.32% floored — are we losing almost half the data?

No. Flooring to zero is recovering truth, not losing data. Let's be precise about what "44.32% floored" means:

For these 640,425 entries, the observed allele fraction was smaller than the expected ambient contribution at that locus: X_obs < α·μ[l]. The entire signal is explained by background noise. The true allele fraction is zero — the cell has no real alternative allele signal here.

The "data" we are setting to zero was not signal in the first place — it was noise that looked like signal. Setting it to the correct value (zero) is not loss; it is accuracy.

Before floor: X = 0.003 (looks like weak B signal) After floor: X = 0.000 (correct — this is entirely ambient) We removed noise, not signal.

The 55.68% that survive correction are the entries with genuine allele signals above the ambient threshold. These drive the clustering.

▶ #12 — Why is the floor% the same across all 4 donors (44.19–44.38%)? Shouldn't different genotypes give different rates?

This is a deep and important question. The answer reveals why the global ambient model is correct.

Floor condition: X[l,n] < α·μ[l]. The right-hand side (soup threshold) is the same for every cell at every locus — it depends on α (global) and μ[l] (per-locus, computed from all singlets, not donor-specific).

The left-hand side (X[l,n]) depends on the cell's genotype. For different donors, different loci will have different genotypes. But on average, the fraction of (cell, locus) pairs where X_obs < soup threshold is the same for every donor, because:

1. Every donor has roughly similar proportions of AA, AB, and BB loci across 64,496 positions.
2. The ambient threshold α·μ[l] is symmetric — it doesn't target any specific donor's allele.

If the floor% varied strongly by donor (>2 pp), it would suggest the ambient model is biased — perhaps the ambient pool comes disproportionately from one donor. The near-identical rates (≤0.2 pp variation) prove the global model is valid here.

— SILHOUETTE —
▶ #13 — The silhouette is negative. Does that mean souporcell assigned cells to the wrong donors?

No. Silhouette negativity here is entirely due to data sparsity, not incorrect clustering.

The silhouette formula computes Euclidean distances between cells in 64,496-dimensional space. With 99.51% sparsity, two cells from the same donor typically have reads at completely different loci. Their Euclidean distance is large — dominated by the positions where one has reads and the other doesn't.

Two cells from different donors have the exact same problem. Their distances are also large and also dominated by non-overlapping coverage. The result: a(i) ≈ b(i) ≈ 0.41, giving s(i) ≈ −0.003 essentially by random chance.

Cell i (Donor 0) and Cell j (Donor 0): Shared covered loci ≈ 0.49% × 0.49% × 80,712 ≈ 2 loci in common Euclidean distance dominated by 400+ non-shared positions = large, random Same calculation for Cell i (Donor 0) and Cell k (Donor 1): identical problem

souporcell uses a probabilistic genotype model, not Euclidean distance, for clustering. Its output is correct. The silhouette is simply the wrong metric for this data.

▶ #14 — If silhouette doesn't work here, why do we compute it at all?

Three reasons:

1. Completeness. Silhouette is the most widely used cluster quality metric. Any reviewer would ask about it. We compute it so we can give a definitive answer: "silhouette = −0.002560, and here is the mathematical proof of why that is expected and correct."

2. Proof of Step 14. The simulation shows silhouette Δ ≈ 0 even at 50% contamination. This is a surprising result that we then explain with the δ_l cancellation proof. If we hadn't computed it, we couldn't demonstrate this insight.

3. Donor 3 is interesting. Donor 3 is the only cluster with positive silhouette (+0.001330). This suggests Donor 3's genotype is more distinct from the population pool mean than the other three donors. This is a real biological signal hidden in an otherwise uninformative metric.

— DAVIES-BOULDIN —
▶ #15 — Why does DB improve when silhouette doesn't? They both measure cluster quality.

They measure cluster quality in fundamentally different ways:

Silhouette asks: "Is each individual cell closer to its own cluster than to others?" This requires computing N × N pairwise distances. At 99.5% sparsity, most pairs share no covered loci → distances are random → silhouette gives noise.

Davies-Bouldin asks: "Are cluster centroids far apart relative to cluster scatter?" It computes 4 centroids + 4 scatter values, then 6 centroid-pair distances. Centroids average over hundreds of cells, so the sparsity noise cancels in the average.

Silhouette: computes ~12.4M pairwise distances → noise dominates DB: computes 4 centroids + 6 centroid pairs → averages reduce noise

Correction reduces within-cluster scatter (sₖ) by 0.000887. This is small but detectable in the DB formula because the averaging in centroid computation amplifies the signal relative to noise. DB sees it; silhouette cannot.

— SIMULATION —
▶ #16 — We inject α_sim and subtract α. Aren't we just undoing ourselves?

This is the most common confusion about the simulation. We are not injecting and subtracting the same amount.

We inject at level α_sim (variable: 1%, 5%, 10%, 20%, 30%, 50%) but correct at level α = 0.01262 (fixed: the estimated true contamination). These are different values in all cases except when α_sim = 0.01262.

Example: α_sim = 10%, α = 1.262% Inject: add 0.10 × μ[l] to every entry Correct: subtract 0.01262 × μ[l] from every entry Residual: (0.10 − 0.01262) × μ[l] = 0.0874 × μ[l] ← not zero

What we are actually testing: "If the real contamination were α_sim but our correction only knows about α = 0.01262, how large would the residual error be?" This is exactly the real-world situation where α is estimated and might be slightly wrong.

The simulation is a robustness test, not an undo operation.

▶ #17 — Why is floor_pct = 0% during the simulation even though it's 44.32% in the real data?

In the real data, the floor condition is: X_raw[l,n] < α·μ[l]. Many low-coverage entries satisfy this because X_raw is already small.

In the simulation, we inject additional contamination before applying the correction:

X_inj = X_raw + α_sim × μ[l] ← injection ADDS to X_raw

Since α_sim ≥ 0.01 and μ[l] ≥ 0, X_inj ≥ X_raw for every entry. An entry that was barely above the floor threshold in X_raw is now pushed higher by the injection. An entry that was below the threshold in X_raw is now above it (or at it).

The result: no entry satisfies X_inj < α·μ[l] because X_inj = X_raw + α_sim·μ[l] ≥ α·μ[l] when α_sim ≥ α (which is true for all tested levels).

This is actually good news: it means the simulation RMSE linearity holds without any flooring interference, making R² = 1.000 mathematically exact.

— RMSE LINEARITY —
▶ #18 — The error simplifies to (α_sim − α)·μ[l]. Why does X_raw cancel?

Let's trace through the algebra carefully for one entry:

X_inj = X_raw + α_sim·μ[l] (injection) X_rec = max(0, X_inj − α·μ[l]) (correction) = X_raw + α_sim·μ[l] − α·μ[l] (no flooring in simulation) = X_raw + (α_sim − α)·μ[l] Error = X_inj − α_sim·μ[l] − X_rec (how far from true value?) = (X_raw + α_sim·μ) − α_sim·μ − (X_raw + (α_sim−α)·μ) = X_raw + α_sim·μ − α_sim·μ − X_raw − (α_sim−α)·μ = −(α_sim−α)·μ ... wait, let's redo: = X_raw − X_raw − (α_sim−α)·μ + 0 = (α_sim − α) · μ[l]

X_raw appears in both X_inj and X_rec and cancels perfectly. The error depends only on the mismatch (α_sim − α) and the locus-specific pool frequency μ[l]. This is why the system behaves linearly — the data itself (X_raw) has no effect on the error structure.

▶ #19 — What does slope = √mean(μ²) = 0.3875 physically mean?

The slope encodes how much genetic diversity exists in the donor pool at the sequenced SNP positions.

√mean(μ²) = RMS of the soup vector. Think of it as: "on average, how different are the donors' allele frequencies from zero?"

If all loci had μ[l] = 0.5 (every donor is heterozygous at every position), slope = 0.5. If all loci had μ[l] = 0 (no variation), slope = 0 and contamination wouldn't matter.

Our slope = 0.3875 means this donor pool has moderate genetic diversity. The fact that RMS(μ) = 0.3875 < mean(μ) = 0.4122 tells you the distribution of μ[l] is right-skewed — most loci have μ ≈ 0 (reference-dominant), pulling the RMS down below the mean.

RMS(μ) = 0.3875 ← affected by the many zero-μ loci mean(μ) = 0.4122 ← only averaged over non-zero loci in the correction

This slope is a property of the data, not a parameter we chose. The fact that the OLS slope (0.3875) matches the theoretical √mean(μ²) (0.3875) is independent confirmation that the math is correct.

— OLS —
▶ #20 — Why do we force the regression through the origin? Normal OLS has an intercept.

We force through the origin because of a physical constraint: when α_sim = 0 (no contamination injected), RMSE must equal exactly 0. There is nothing to measure, nothing to recover, no error.

When α_sim = 0: X_inj = X_raw + 0·μ = X_raw (no change) X_rec = max(0, X_raw − α·μ) ≠ X_raw ← wait... Actually RMSE → 0 only in the limit α_sim → α → 0

More precisely: RMSE = |α_sim − α| × RMS(μ). When α_sim = α = 0.01262, RMSE = 0. When α_sim → 0, RMSE → α × RMS(μ) = 0.01262 × 0.3875 ≈ 0.00489 — not quite zero.

In our OLS fit, we treat α ≈ 0 (a valid approximation since α = 1.26% ≪ 1), which shifts the zero-crossing from α_sim = α to α_sim = 0. The tiny offset (0.00489) is absorbed into the slope estimate, giving b̂ = 0.3875 instead of 0.3870. This is why the fit is "through the origin" in an approximate sense.

— R² —
▶ #21 — R² = 1.000 sounds too good. How can any real experiment achieve perfect fit?

It is not a coincidence or an error. R² = 1.000 is an algebraic consequence of the derivation, not an empirical finding.

From the derivation: error per entry = (α_sim − α) × μ[l]. This is a deterministic function of known quantities. There is no random noise in this expression — for non-floored entries, the error is exactly predictable from α_sim, α, and μ[l].

If error = (α_sim − α) × μ[l] exactly, then RMSE = √mean(error²) = |α_sim − α| × √mean(μ²) exactly, and the line RMSE = slope × α_sim fits perfectly.

The reason floor_pct = 0% in the simulation is crucial here. If entries were floored, the error formula would break down for those entries (flooring is non-linear). With no flooring, every entry follows the exact linear formula, giving R² = 1.000000 with residuals at 10⁻⁵ scale (floating-point rounding only).

— SILHOUETTE Δ≈0 —
▶ #22 — If ambient RNA shifts every cell's X values, why don't distances between cells change?

Because the shift is uniform across all cells at the same locus. When you subtract one cell's value from another's, the shift cancels.

Before: distance at locus l = |X[l,i] − X[l,j]| Shift: δ_l = (α_sim − α) × μ[l] ← same for cell i AND cell j After: distance = |X[l,i] + δ_l − X[l,j] − δ_l| = |X[l,i] − X[l,j]| ← δ_l cancels exactly

It's like adding the same constant to every temperature reading in a room. The temperature differences between thermometers don't change. The absolute values change, but all the differences — which is what distances measure — are unchanged.

▶ #23 — But the simulation shows Sil Δ = −0.000087 at 50%, not exactly zero. Why?

Two effects break the perfect cancellation:

1. Flooring asymmetry. A small number of entries get floored during simulation (when α_sim is large). A floored entry has X_corr = 0 regardless of the shift — it no longer contributes symmetrically to distances. This creates a tiny non-zero Δ that grows with α_sim.

2. Sampling noise. The silhouette is computed on a subsample of ≤150 cells per cluster (not all 3,522). Different random samples give slightly different silhouette values. The variation across runs is on the order of 10⁻⁴ — which is the same magnitude as the observed Δ.

At α_sim = 50%: Theoretical Δ = 0 (exact, for non-floored entries) Observed Δ = −0.000087 ← sampling noise + flooring Magnitude: 0.000087 / 0.002560 = 3.4% of the silhouette value

Both effects are negligible at the contamination levels tested. The observed Δ confirms, not contradicts, the theory.

— DIMENSIONALITY REDUCTION —
▶ #24 — Why can't we just plot the raw allele fractions in 2D directly without PCA?

Because we have 64,496 dimensions. To plot in 2D, we must reduce. The options are:

Option 1: Pick 2 loci arbitrarily. Any two loci would show you one dot per cell — but 99.5% of cells have NaN at any given locus. You'd mostly see missing data.

Option 2: t-SNE / UMAP directly on 64,496 features. Computationally infeasible. t-SNE is O(N log N) in features. On 64,496 features × 3,522 cells: hours to days of compute.

Option 3 (what we do): PCA first. PCA compresses 64,496 → 50 dimensions in ~20 seconds, retaining ~95% of variance. Then t-SNE on 50 dimensions takes ~47 seconds. Total: ~1 minute for a meaningful 2D visualisation.

Direct t-SNE on 64,496D: O(3522 × 64,496 × log(3522)) ≈ hours PCA + t-SNE: O(3522 × 64,496) for PCA + O(3522 × 50) for t-SNE ≈ 1 min
▶ #25 — In t-SNE, the distances between clusters don't mean anything. So what does the plot actually show?

This is a critical warning about t-SNE interpretation. In a t-SNE plot:

Meaningful: Whether two cells are in the same cluster (close together) or different clusters (far apart within the 2D layout). Cluster shapes are reliable.

NOT meaningful: The actual distance between cluster centres. Two clusters that appear far apart in t-SNE may actually be close in high-dimensional space, and vice versa. The optimization explicitly distorts long-range distances to make local structure visible.

t-SNE preserves: local neighbourhood structure t-SNE distorts: global distances between clusters

UMAP is better in this regard — it preserves both local structure and a reasonable approximation of global distances. For soupXcell, both methods serve the same purpose: show that 4 donor clusters exist and remain separable before and after correction.

— X_TRUE —
▶ #26 — You keep saying X_true but we never measure it. Is it a theoretical construct or a real number?

It is both. X_true is:

Theoretically: The allele fraction a cell would produce at locus l if zero ambient RNA existed. Determined entirely by the cell's diploid genotype at that position: 0.0 (AA), ~0.5 (AB), or 1.0 (BB).

Practically: Inferred from souporcell's clustering. Once souporcell assigns a cell to Donor 2 and determines that Donor 2 is reference-homozygous at locus l, we know X_true[l, this_cell] = 0.0. Not measured — inferred from the cluster assignment and the per-donor genotype.

X_obs[l,n] = what ref.mtx + alt.mtx give us ← measured X_corr[l,n] = max(0, X_obs − α·μ[l]) ← computed X_true[l,n] = 0.0 / 0.5 / 1.0 based on genotype ← inferred Goal: X_corr → X_true (correct estimate of true signal)

This is also why the simulation is necessary for validation — by injecting known contamination on top of X_raw, we create a situation where the "ground truth" is X_raw (before injection), and we can check how well we recover it.

▶ #27 — Can X_corr be negative or greater than 1? What happens at the extremes?

Can it be negative? Without max(0,·), yes. This is why the floor exists. After flooring, X_corr ∈ [0, 1] always.

Can it be greater than 1? X_corr = X_obs − α·μ[l]. Since α·μ[l] ≥ 0, X_corr ≤ X_obs ≤ 1. So X_corr ≤ 1 always. The correction can only decrease values, never increase them above 1.

X_obs ∈ [0, 1] (by definition: A/(R+A) ∈ [0,1]) soup[l] = α·μ[l] ≥ 0 X_corr = max(0, X_obs − soup[l]) → X_corr ∈ [0, X_obs] ⊆ [0, 1] always

The floor handles one edge case; the upper bound is guaranteed by the formula structure. soupXcell preserves the physical constraint that allele fractions must lie in [0, 1].

— PRACTICAL —
▶ #28 — What if α was estimated incorrectly by consensus.py? How wrong could the correction be?

The simulation directly answers this. RMSE = |α_sim − α| × RMS(μ) = 0.3875 × |α_sim − α|.

If the true contamination is 2% but we estimated 1.262%, the mismatch is 0.738 pp. Error per entry ≈ 0.00738 × 0.3875 ≈ 0.003 allele fraction units. Tiny.

α_true = 0.020, α_estimated = 0.01262 Mismatch = 0.0074 Error per entry ≈ 0.0074 × 0.3875 = 0.003 allele fraction units For X_corr ≈ 0.5 (heterozygous), this is 0.6% of the signal

The correction is highly robust to α mis-estimation because α is small. Doubling α from 1.26% to 2.52% only doubles the error — which was already negligible. The correction would need to be off by a factor of 10× (α_estimated = 0.126 when true is 0.013) to produce a material effect.

▶ #29 — The within-cluster std improved by only −0.43%. Is that really a meaningful improvement?

The absolute improvement (−0.000887) is small, but it must be understood in context:

1. α = 1.262% is a mild contamination level. The correction removes ~1.26% of each allele fraction. A −0.43% reduction in within-cluster std is exactly proportional to this — it is neither over- nor under-correcting. This is the expected effect, not a failure.

2. Downstream analyses benefit cumulatively. In a multi-sample integration or differential expression analysis, cells from the same donor appearing more consistent reduces false positives. A 0.4% improvement in every cell × 3,523 cells = meaningful aggregate effect.

3. The simulation proves the method is exact at all contamination levels. If α were 10%, the improvement would be ~3.4%. If α were 20%, ~6.9%. The correction scales correctly; the small improvement here simply reflects small contamination.

At α = 0.01262: Δσ = −0.000887 (−0.43%) At α = 0.10: Δσ ≈ −0.007 (−3.4%) [theoretical] At α = 0.20: Δσ ≈ −0.014 (−6.9%) [theoretical]
▶ #30 — Could we skip dimensionality reduction and just look at allele fractions directly to evaluate clusters?

In principle yes, but practically no. The allele fraction matrix is 64,496 × 3,522 — you cannot look at it directly. Any scalar summary (like silhouette or DB) loses information. Dimensionality reduction is the only way to see the cluster structure.

There is also a theoretical argument: the 2D t-SNE / UMAP embedding preserves neighbourhood structure from the 50-dimensional PCA space, which in turn approximates the dominant variation in the 64,496-dimensional allele fraction space. A 2D scatter plot of t-SNE coordinates communicates more about donor separation than any scalar metric.

The before/after t-SNE comparison provides visual confirmation that:

(1) 4 donor clusters exist and are separable (2) Cluster positions are preserved after correction (δ_l cancels) (3) Cluster shapes are slightly tighter after correction (σ_within ↓) (4) No cluster merging or splitting occurred

Visual validation complements, rather than replaces, the quantitative metrics (DB, within-cluster std, R²).
